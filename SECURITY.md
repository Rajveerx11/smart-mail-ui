# Security Policy

## Supported Versions

Security updates are generally applied to the default branch ($branch). If stable release branches exist, they may also receive critical fixes.

| Version | Supported |
| ------- | --------- |
| main | Yes |
| older branches | Best effort |

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Instead, report privately using one of these options:

1. GitHub Security Advisories (preferred) for this repository.
2. Directly contact the repository maintainer via GitHub profile.

Include the following in your report:

- Clear description of the issue
- Steps to reproduce
- Proof-of-concept or sample payload (if safe)
- Potential impact
- Suggested remediation (optional)

## Response Timeline

- Initial acknowledgement: within 3-7 days
- Triage and severity assessment: within 7-14 days
- Fix timeline: based on severity and complexity

## Disclosure Guidelines

- Allow maintainers reasonable time to investigate and patch before public disclosure.
- Avoid sharing exploit details publicly until a fix is available.

## Scope

This policy applies to source code, dependencies, CI/CD workflows, and deployment configuration within **smart-mail-ui**.

## Safe Harbor

Good-faith security research that avoids privacy violations, service disruption, and data destruction will be treated as responsible disclosure.
